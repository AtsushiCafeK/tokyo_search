# さくらのレンタルサーバ等へデプロイ（公開）する手順

現在のアプリは **Next.js** で作られており、初期状態では「Node.jsが動くサーバー」が必要です。
しかし、「さくらのレンタルサーバ」や「エックスサーバー」のような、一般的な（HTMLファイルを置くだけの）Webサーバーで動かすためには、**アプリを静的なHTML/CSS/JSファイルに書き出す（Static Export）** 必要があります。

今回のアプリは、1ページ完結の一体感を維持しつつ、リンクを別タブで開くために **Google Programmable Search Engine (埋め込みウィジェット)** を採用しています。

この方式で動かすためには、Googleから「検索エンジンID」を取得する必要があります。

---

## ステップ 0: 検索エンジンID（CX）の取得

1. [Google Programmable Search Engine](https://programmablesearchengine.google.com/about/) にアクセスします。
2. 検索対象サイトとして `city.hachioji.tokyo.jp` と `metro.tokyo.lg.jp` を登録します。
3. 発行された **検索エンジンID (例: `a1b2c3d4e5f6g7h8i`)** をコピーします。

## ステップ 1: 環境変数の設定

プロジェクト内にある `.env.local` という名前の新しいファイルを作成し（既にある場合は開き）、以下のように記載してください。

```env
NEXT_PUBLIC_GOOGLE_CX=取得した検索エンジンID
```

※ローカルでの開発・検証時にはこの設定が必要です。
※**本番サーバーへのアップロード前**にこの設定を行った状態でビルドを行ってください。

## ステップ 2: Next.js の設定を確認する

まず、Next.jsに「静的なファイルとして書き出す」ように指示をします。
プロジェクト内にある `next.config.ts` を開き、以下のように `output: 'export'` という設定を追加してください。

```typescript
import type { NextConfig } from "next";

const nextConfig: NextConfig = {
  /* config options here */
  output: 'export', // ← この行を追加します
  
  // (オプション) 画像最適化APIは静的エクスポートでは使えないため無効化します
  images: {
    unoptimized: true,
  }
};

export default nextConfig;
```

## ステップ 2: デプロイ用のファイルを生成する（ビルド）

ターミナルを開き、プロジェクトのフォルダ（`gov_hachiouji`）で以下のコマンドを実行します。

```bash
npm run build
```

このコマンドを実行すると、プログラムが自動で解析され、プロジェクト内に **`out`** という新しいフォルダが作成されます。

> **重要:** この `out` フォルダの中身すべてが、「公開用（本番用）のホームページデータ」になります。

## ステップ 3: サーバーへアップロードする

1. お使いのFTPソフト（FFFTP、FileZilla、Cyberduckなど）を開き、さくらのレンタルサーバ等に接続します。
2. サーバー上の公開したい場所（例: `www` ディレクトリや `public_html` ディレクトリ内の任意のフォルダなど）を開きます。
3. 先ほど生成された **`out` フォルダの中身すべて** を、対象のディレクトリにそのままアップロード（ドラッグ＆ドロップ）します。

※ `out` フォルダ自体ではなく、`out` フォルダの中に入っている `index.html` などを直接アップロードしてください。

## ステップ 4: ブラウザで確認する

アップロードしたURL（例: `http://あなたのドメイン.com/`）にブラウザからアクセスし、正しく表示・検索が動くことを確認して完了です。

---

> [!TIP]
> **💡 ヒント : サブディレクトリで公開する場合**
> もし `http://あなたのドメイン.com/search-app/` のように、サブディレクトリ（フォルダの下）に公開したい場合は、画像のパスなどがズレないように `next.config.ts` にプレフィックスの設定を追加する必要があります。
> その場合は、以下の設定を `output: 'export'` と一緒に追加してからビルドしてください。
> ```typescript
>   basePath: '/search-app', // あなたの公開先フォルダ名
> ```
